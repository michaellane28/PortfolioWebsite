A dynamic web application that highlights NHL games based on player and team criteria, updated daily with current matchups

Games are filtered for each day based on a specific criteria: One team must have been top 5 in penalties per games in the previous season, and the other team must have two or more players who are both the first forward line and the first powerplay line.

This project has two main Python files:

get_games_for_day.py:

Uses Selenium to scrape NHL team line combinations and powerplay units from a specific website.

Loads the NHL 2024-25 schedule from an Excel file and filters games based on the current date.

Matches NHL teams in the top 5 list against their opponents, identifying players on both the first line and powerplay unit.

Appends fitting games to a separate excel file with relevant information

Configures Selenium to run in a headless Chrome browser for automated, non-visual web scraping.

update_dates.py:

This script updates the dates for upcoming games and refreshes the data in the application.

Reads and writes content to a script.js file, enabling automated updates to specific JavaScript date variables.

Utilizes regular expressions to find the maxDate and currentDate variables within the JavaScript code.

Automatically increments the maxDate and currentDate by one day using Python's datetime module.

Overwrites the original script.js file with the updated date values, ensuring the script remains current.


This project also uses two Github Actions:

update_schedule.yml:

Executes daily at 12:00 am CDT and can be manually triggered.

Runs get_games_for_day.py to update the filtered schedule excel file.

Configures Git, commits updates to filtered_schedule.xlsx, and pushes to the repository.

update_dates.py:

Executes daily at 12:10 CDT and can be manually triggered.

Runs update_dates.py to increment dates in script.js.

Configures Git, commits updates to script.js, and pushes to the repository.
